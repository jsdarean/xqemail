# 需求提取kimi版 - Chrome 扩展

## 功能简介

1. **自动提取**：从当前页面提取需求信息（编号、名称、提出人、背景及目标、描述）
2. **邮件发送**：通过本地 SMTP 中继服务器直接发送邮件，支持附件和 IMAP 已发送存档
3. **收件人管理**：支持手动添加、Excel 导入/导出联系人
4. **规则可配**：自定义 CSS 选择器和页面匹配规则

---

## 安装方法

### 第一步：安装 Node.js（必需）

本扩展依赖本地 Node.js 中继服务器，请先安装 Node.js（推荐 v18+）：

- 官网下载：https://nodejs.org/
- 安装后打开命令行，输入 `node -v` 确认安装成功

### 第二步：安装中继服务器依赖

打开命令行，进入 `local-smtp-server` 目录，执行：

```bash
cd local-smtp-server
npm install
```

### 第三步：加载 Chrome 扩展

1. 打开 Chrome 地址栏输入：`chrome://extensions/`
2. 开启右上角 **「开发者模式」**
3. 点击 **「加载已解压的扩展程序」**
4. 选择 `requirement-email-extractor` 文件夹

---

## 启动中继服务器

每次使用扩展前，需要先启动本地邮件中继服务器：

**方式一（推荐 - 后台静默运行）：**
双击 `local-smtp-server/start-silent.vbs`

**方式二（可见窗口，方便查看日志）：**
双击 `local-smtp-server/start-server.bat`

---

## 配置邮箱

1. 点击扩展图标 → **「📧 邮箱设置」**
2. 填写你的 SMTP 服务器信息（联系邮箱管理员获取）：
   - SMTP 服务器地址（如 `smtp.company.com`）
   - 端口（通常是 465 或 587）
   - 邮箱账号和授权码/密码
3. （可选）填写 IMAP 服务器地址，用于将发送记录存档到「已发送」文件夹
4. （可选）设置邮件签名和默认抄送邮箱
5. 点击 **「💾 保存邮箱设置」**

---

## 使用说明

### 导入收件人

1. 切换到 **「👥 收件人管理」** 标签
2. 手动添加，或点击 **「📥 导入 Excel 文件」** 导入 .xlsx 文件
3. Excel 需包含列：姓名、系统、邮箱

### 提取并发送

1. 打开需求页面，点击扩展图标
2. 点击 **「🔍 提取需求信息」**
3. 勾选收件人，填写抄送（可选）
4. （可选）添加附件：点击 **「➕ 添加附件」**
5. 点击 **「📧 直接发送邮件」**

## 安全说明

- SMTP 密码和数据库密码在保存时会使用 AES-256-GCM 加密后再写入 Chrome 本地存储。
- 本地中继服务器仅监听 `127.0.0.1:2525`，外部网络无法访问。
- 中继服务器已对敏感接口启用来源校验，仅允许 Chrome 扩展（`chrome-extension://`）调用发件、写库等接口。

## 常见问题

**Q: 点击发送提示「本地邮件服务器未启动」？**
A: 请先双击 `start-server.bat` 或 `start-silent.vbs` 启动中继服务器。

**Q: 启动服务器时提示端口被占用？**
A: 打开命令行执行 `netstat -ano | findstr :2525` 找到占用进程，再执行 `taskkill /PID <PID> /F` 结束它，然后重新启动。

**Q: 发送失败？**
A: 检查 SMTP 配置是否正确（服务器地址、端口、账号、授权码）。可在 `start-server.bat` 窗口中查看错误日志。

**Q: 网页邮箱看不到已发送记录？**
A: 需要在邮箱设置中配置 IMAP 服务器地址。

**Q: 收件人显示为「本地」而不是「数据库」？**
A: 请在「邮箱设置」的 MySQL 数据库设置中填写正确的连接信息并保存，插件会自动尝试从 `sa_info` 表加载联系人。
